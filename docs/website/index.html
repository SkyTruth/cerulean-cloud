Public API

Cerulean API Guide

Cerulean exposes an OGC API Features-compatible TiPG endpoint at https://api.cerulean.skytruth.org/. This guide mirrors the workflows in notebooks/Cerulean API Guide.ipynb and focuses on query patterns for slick detections, source attribution, AOIs, and human-reviewed classes.

Base URL

https://api.cerulean.skytruth.org/

Main pattern

GET /collections/{collectionId}/items

Primary format

GeoJSON FeatureCollection

For deeper API semantics, see TiPG docs and CQL-2 syntax reference.

For map-driven exploration, use the Cerulean UI: https://cerulean.skytruth.org/.

Quickstart

Start with a spatial query against public.slick_plus. This returns slick detections in part of the Mediterranean Sea. 

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1' \
  -H 'Content-Type: application/json;'
import requests

url = (
    "https://api.cerulean.skytruth.org/collections/public.slick_plus/items"
    "?limit=100"
    "&bbox=10.9,42.3,19.7,36.1"
)

data = requests.get(url, headers={"Content-Type": "application/json;"}).json()
print("Matched:", data.get("numberMatched"))
print("Returned:", data.get("numberReturned"))

Pagination defaults to limit=10 when omitted (max limit=9999). Use offset for paging through large result sets.

Query Model

Cerulean follows the TiPG + OGC API Features model. Most operations are GET queries with URL parameters.

Parameter Type Usage
bbox minLon,minLat,maxLon,maxLat Spatial filter.
datetime start/end (UTC) Temporal window, ISO format YYYY-MM-DDTHH:MM:SSZ.
limit integer Rows returned. Notebook examples use 100; default is 10, max is 9999.
offset integer Pagination offset.
sortby string Sort field; prefix - for descending.
filter CQL-2 expression Property filters using operators like GT, GTE, EQ, and IS NULL.
f string Output format, e.g. geojson or csv.
id, s1_scene_id, hitl_cls collection-specific filters Direct record lookup and common slick filtering dimensions.
source_type, mmsi_or_structure_id collection-specific filters Used heavily with public.source_plus.
aoi_id, collation_threshold collection-specific filters Used by public.get_slicks_by_aoi.

URL-encode CQL-2 filters when using spaces/operators. In browser query strings, expressions like GT, GTE, and IS NULL should be encoded.

Responses

Most collection item requests return a GeoJSON FeatureCollection.

{
  "type": "FeatureCollection",
  "numberMatched": 1234,
  "numberReturned": 100,
  "features": [
    {
      "type": "Feature",
      "id": 3582918,
      "geometry": { "...": "..." },
      "properties": { "...": "..." }
    }
  ]
}
  • numberMatched is total rows matching filters (independent of limit).
  • numberReturned is the number included in the current page.
  • For single-ID style lookups, you still receive a feature collection in most collection queries.

Collections Reference

The notebook primarily queries the collections below through /collections/{collectionId}/items.

public.slick_plus

Slick detection geometries and metadata, including source-collation scores and human-review labels.

GET /collections/public.slick_plus/items
Field Description
id Unique slick record identifier.
geometry Slick footprint polygon/multipolygon.
s1_scene_id Sentinel-1 scene identifier.
slick_timestamp UTC time associated with detection.
machine_confidence Model confidence proxy.
area, perimeter, length Size and shape metrics (meters / square meters).
linearity, fill_factor, aspect_ratio_factor Shape descriptors used in interpretation workflows.
centerlines Derived line geometry representation of slick extent.
max_source_collated_score Highest source-attribution score for the slick.
aoi_type_1_ids, aoi_type_2_ids, aoi_type_3_ids Intersecting AOI IDs for EEZ / IHO / MPA categories.
source_type_1_ids, source_type_2_ids, source_type_3_ids Nearby source identifiers for vessel / infrastructure / dark vessels.
hitl_cls / hitl_cls_name Human-in-the-loop label and class name (nullable).
slick_url Direct Cerulean UI link for visual inspection.

Notebook guidance: max_source_collated_score > 0 is a practical threshold for higher-credibility detections prior to manual review.

public.source_plus

Potential source attributions for slicks (vessel, infra, dark, natural).

GET /collections/public.source_plus/items
Field Description
id Unique source-plus record identifier.
geometry Slick geometry associated with the source candidate.
mmsi_or_structure_id Vessel MMSI or infrastructure structure identifier.
source_type One of VESSEL, INFRA, DARK, NATURAL.
slick_confidence Detection-confidence proxy retained with the attribution record.
source_collated_score Relative attribution score, often filtered with > 0.
source_rank Ranking of candidate source per slick.
git_tag Scoring/model compatibility marker.
slick_url / source_url UI links for investigating slicks and sources.

public.aoi

Canonical Area of Interest registry across EEZ, IHO, and MPA categories.

GET /collections/public.aoi/items

public.aoi_eez

EEZ AOI lookup table with mrgid to aoi_id mapping.

GET /collections/public.aoi_eez/items

public.aoi_iho

IHO Sea Area AOI lookup table.

GET /collections/public.aoi_iho/items

public.aoi_mpa

Marine Protected Area AOI lookup table (supports WDPA-based lookup).

GET /collections/public.aoi_mpa/items

public.cls

Human review class dictionary and hierarchy (supercls relationships).

GET /collections/public.cls/items

public.get_slicks_by_aoi

Convenience collection for slick retrieval by aoi_id plus a configurable collation threshold.

GET /collections/public.get_slicks_by_aoi/items

Notebook Query Examples

These map directly to the examples from Cerulean API Guide.ipynb and are formatted for production usage.

Example 1: Return slicks within a bounding box

GET /collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1 
curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1' \
  -H 'Content-Type: application/json;'

Example 2: High-grade results and filter by datetime

Sort by highest max_source_collated_score in a date window.

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1&datetime=2024-01-01T00:00:00Z/2025-01-01T00:00:00Z&filter=max_source_collated_score%20%3E%200&sortby=-max_source_collated_score' \
  -H 'Content-Type: application/json;'

Example 3: Other basic filtering (CQL-2)

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1&datetime=2024-01-01T00:00:00Z/2025-01-01T00:00:00Z&sortby=slick_timestamp&filter=max_source_collated_score%20GTE%200.0%20AND%20area%20GT%205000000' \
  -H 'Content-Type: application/json;'

Example 4: Filter by nearby vessel source

Use NOT source_type_1_ids IS NULL to return slicks with vessel candidates.

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1&datetime=2024-01-01T00:00:00Z/2025-01-01T00:00:00Z&sortby=slick_timestamp&filter=max_source_collated_score%20GTE%200.0%20AND%20area%20GT%205000000%20AND%20NOT%20source_type_1_ids%20IS%20NULL' \
  -H 'Content-Type: application/json;'

Example 5: Download GeoJSON and CSV

If your filter contains spaces/symbols, URL-encode the query string before downloading. 

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1&sortby=slick_timestamp&datetime=2024-01-01T00:00:00Z/2025-01-01T00:00:00Z&filter=max_source_collated_score%20GTE%200.0%20AND%20area%20GT%205000000%20AND%20NOT%20source_type_1_ids%20IS%20NULL' \
  -o example_5.geojson
curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?limit=100&bbox=10.9,42.3,19.7,36.1&sortby=slick_timestamp&datetime=2024-01-01T00:00:00Z/2025-01-01T00:00:00Z&filter=max_source_collated_score%20GTE%200.0%20AND%20area%20GT%205000000%20AND%20NOT%20source_type_1_ids%20IS%20NULL&f=csv' \
  -o example_5.csv
python -c "import urllib.parse; print(urllib.parse.quote('https://api.cerulean.skytruth.org/collections/public.slick_plus/items?filter=max_source_collated_score GTE 0.0 AND area GT 5000000', safe=':/?=&'))"

Example 6: Return a specific slick by ID

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?id=3582918' \
  -H 'Content-Type: application/json;'

Example 7: Return all slicks from a Sentinel-1 scene

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?s1_scene_id=S1A_IW_GRDH_1SDV_20240113T165611_20240113T165636_052091_064BBD_918F' \
  -H 'Content-Type: application/json;'

Example 8: Filter by EEZ, IHO, or MPA (AOI workflow)

Step 1: Find AOI records by name.

curl --globoff \
  "https://api.cerulean.skytruth.org/collections/public.aoi/items?filter=LOWER(name)%20LIKE%20%27%25greek%25%27"

Step 2: Resolve AOI by EEZ MRGID (example: 5679).

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.aoi_eez/items?mrgid=5679'

Step 3: Query slicks using returned aoi_id.

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.get_slicks_by_aoi/items?limit=100&aoi_id=12345&collation_threshold=0'

Example 9: Iterate through source IDs

Return top-ranked source candidates for one vessel/infrastructure identifier. The notebook demonstrates iterating multiple IDs. 

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.source_plus/items?mmsi_or_structure_id=372519000&filter=source_collated_score%20GT%200.0%20AND%20source_rank%20EQ%201' \
  -H 'Content-Type: application/json;'

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.source_plus/items?mmsi_or_structure_id=477932400&filter=source_collated_score%20GT%200.0%20AND%20source_rank%20EQ%201' \
  -H 'Content-Type: application/json;'

Notebook guidance: prefer records with git_tag >= 1.1.0 when comparing source model outputs across versions.

Example 10: Locate potential dark vessel polluters

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.source_plus/items?bbox=10.9,42.3,19.7,36.1&datetime=2025-05-01T00:00:00Z/2025-05-09T00:00:00Z&source_type=DARK&filter=source_collated_score%20GT%200.5&sortby=-source_collated_score' \
  -H 'Content-Type: application/json;'

Tradeoff reminder from the notebook: higher thresholds (for example > 0.5) improve precision but reduce recall.

Example 11: Obtain human reviewed slicks

Query class dictionary first, then return all reviewed records or target a specific class with hitl_cls. 

curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.cls/items' \
  -H 'Content-Type: application/json;'
curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?bbox=10.9,42.3,19.7,36.1&limit=10&filter=NOT%20hitl_cls%20IS%20NULL' \
  -H 'Content-Type: application/json;'
curl --globoff \
  'https://api.cerulean.skytruth.org/collections/public.slick_plus/items?bbox=10.9,42.3,19.7,36.1&limit=10&hitl_cls=8' \
  -H 'Content-Type: application/json;'

Class hierarchy is captured by supercls in public.cls; subclass labels are often more common than their parent class labels.

Interpretation Notes

  • Not all returned geometries are true oil detections. Use slick_url to validate in the UI.
  • max_source_collated_score and source_collated_score are heuristics, not ground truth labels.
  • Notebook recommendation: consider max_source_collated_score > 0 as a practical credibility threshold.
  • HITL-reviewed records are a subset of all detections.

Conclusion

This guide is a practical subset of Cerulean API workflows. For full endpoint discoverability and live schema details, continue in the live API docs.
